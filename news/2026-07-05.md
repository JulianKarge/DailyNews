### Mistral veröffentlicht Leanstral 1.5 für formale Beweise und Code-Verifikation

Mistral beschreibt Leanstral 1.5 als aktualisiertes, offen verfügbares Modell für Lean-4-Proof-Engineering, automatisiertes Theorem Proving und Autoformalization. Die Model Card nennt 119B Gesamtparameter, 6,5B aktive Parameter und bis zu 256k Kontext; Hugging Face führt das Modell unter Apache-2.0 und beschreibt es als open-source Code-Agent-Modell für mathematische Beweise und Software-Spezifikationen.

**Warum das wichtig ist:** Formale Beweise und verifizierbare Spezifikationen rücken näher an praktische Entwickler-Workflows heran. Wenn spezialisierte Modelle Bugs in Code und Eigenschaften von Systemen besser in formale Prüfungen übersetzen können, entstehen neue Bausteine für sicherheitskritische Software, Forschung, automatisierte Reviews und agentische Coding-Pipelines.

### GitHub öffnet Streaming und API für Copilot-Agentensitzungen

GitHub stellt Copilot Agent Session Streaming als Public Preview bereit. Enterprise-Cloud-Kunden mit Enterprise Managed Users können damit Sitzungsdaten aus Copilot-Clients wie GitHub.com-Cloud-Agenten, Copilot CLI, VS Code, Visual Studio und Partner-IDEs abrufen; die Daten umfassen laut GitHub unter anderem Prompts, Antworten und Tool Calls und lassen sich über Streaming-Endpunkte oder eine REST API auswerten.

**Warum das wichtig ist:** Coding-Agenten werden produktiver, aber auch schwerer zu kontrollieren. Sichtbarkeit in Prompts, Antworten und Tool-Nutzung ist ein wichtiger Schritt für Auditierbarkeit, Compliance, Kostenkontrolle und Incident-Analyse, besonders wenn Unternehmen Agenten über mehrere Entwicklungsumgebungen hinweg einsetzen.

**Quellen:** [GitHub Changelog](https://github.blog/changelog/2026-07-02-copilot-agent-session-streaming-is-now-in-public-preview)

**Tags:** GitHub, Copilot, Coding Agents, Governance, Enterprise AI | **Confidence:** high

### pxpipe zeigt Token-Kompression per Bild-Kontext für Claude-Code-Workflows

Das Open-Source-Projekt pxpipe rendert große Textkontexte wie Systemprompts, Tool-Dokumentation und Verlauf lokal als PNGs, bevor Requests an Claude Code weitergegeben werden. Der README beschreibt den Ansatz als lokalen Proxy und nennt deutlich niedrigere Eingabetoken auf dichten Coding-Kontexten; die Projekt-Findings betonen zugleich, dass es sich um eine verlustbehaftete Kompression mit Grenzen bei exakter Textwiedergabe handelt.

**Warum das wichtig ist:** Der Ansatz ist weniger eine fertige Best Practice als ein Signal: Lange Agenten-Kontexte bleiben teuer und technisch fragil. Teams, die Coding-Agenten einsetzen, brauchen Messung, Kostenkontrolle und klare Qualitätsgrenzen, bevor sie Kontext-Kompression oder multimodale Workarounds in produktive Workflows übernehmen.
