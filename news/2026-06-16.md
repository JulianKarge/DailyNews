# Daily AI News — Dienstag, 16.06.2026

Heute im Fokus: GitHub Copilot CLI wird über Slash-Commands greifbarer steuerbar, und ein offener Datensatz soll mehrsprachige Developer-AI verbessern.

## GitHub erklärt zentrale Slash-Commands für Copilot CLI als Steuerungsebene für Terminal-Agenten

GitHub veröffentlicht eine praxisnahe Übersicht zu Slash-Commands in GitHub Copilot CLI. Die Befehle dienen dazu, einen Terminal-basierten AI-Agenten gezielter zu steuern, Kontext zu setzen und typische Aktionen direkt aus der Kommandozeile auszulösen.

### Warum das wichtig ist

Terminal-Agenten werden nützlicher, wenn sie nicht nur freie Chat-Eingaben verstehen, sondern klare Bedienmuster für wiederkehrende Entwickleraufgaben haben. Slash-Commands sind ein einfaches Interface zwischen natürlicher Sprache und kontrollierbaren Workflows — besonders relevant für Teams, die Agenten in bestehende Shell-, Git- und CI-Prozesse einführen wollen.

## GitHub veröffentlicht offenen Datensatz für mehrsprachige Developer-Inhalte

GitHub stellt einen neuen repository-level Datensatz unter CC0-1.0 vor, der Forschenden und Entwicklern helfen soll, mehrsprachige Entwickler-Inhalte zu finden und für LLM- sowie Tooling-Forschung nutzbar zu machen.

### Warum das wichtig ist

Viele Coding- und Dokumentationsmodelle sind noch stark englisch geprägt. Ein offener Datensatz für mehrsprachige Developer-Inhalte kann bessere Evaluierung, Training und Analyse von Werkzeugen fördern, die weltweit nutzbar sein sollen — von Code-Suche und Dokumentation bis zu Assistenzsystemen für internationale Teams.
